# Contributing to DecisionFlow AI

Thank you for your interest in contributing to DecisionFlow AI! We welcome community contributions to help improve the platform.

## Getting Started

1. **Fork the Repository**: Create a personal fork on GitHub.
2. **Clone the Fork**: Clone the project locally:
   ```bash
   git clone https://github.com/Vamshi377/xlventure.git
   ```
3. **Set Up Environments**: Follow the setup instructions in [docs/Installation.md](docs/Installation.md).
4. **Create a Feature Branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Guidelines

* **Code Formatting**: Ensure python code adheres to PEP 8 standards.
* **Component Rules**: Reusable components should be typed with TypeScript in Vite.
* **Testing**: Run local tests and verify the application compiles cleanly (`npm run build`).

## Submitting Pull Requests

1. Commit your changes with descriptive messages.
2. Push your feature branch to your fork.
3. Open a Pull Request from your branch to the `main` repository branch.
4. Follow the PR template guidelines.
